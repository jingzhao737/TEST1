# 图片目录结构

images/
├── README.md           ← 你正在看的总说明
├── works/              ← 作品列表 hover 预览图（6 张）
│   └── README.md       ← 详细替换说明
├── showcase/           ← 视差展示区大图（3 张）
│   └── README.md       ← 详细替换说明
└── detail/             ← 作品详情页图片（18 张：6 hero + 12 gallery）
    └── README.md       ← 详细替换说明

============================================
快速开始：替换你的图片
============================================

1. 准备你的作品图片
2. 按对应文件名命名（见各子文件夹 README.md）
3. 放入对应子文件夹，覆盖现有文件
4. 刷新浏览器即可

============================================
所有引用图片的文件位置
============================================
index.html:
  - data-image="images/works/xxx-preview.jpg"   (6 处，卡片的 hover 预览)
  - data-hero="images/detail/xxx-hero.jpg"      (6 处，详情页顶部大图)
  - background-image:url(images/showcase/...)   (3 处，视差面板背景)

app.js:
  - workData.xxx.gallery: [...]                 (12 处，详情页双图)

============================================
图片不显示？
============================================
- 确认文件名和扩展名完全一致（区分大小写）
- 确认文件放在正确的子文件夹中
- 打开浏览器开发者工具 (F12) → Network 标签页，
  刷新页面查看哪些图片返回 404

============================================
前端自动压缩建议
============================================
- 作品预览图建议 WebP 格式，600×440px
- Showcase 大图建议 WebP 格式，1920×1200px
- Detail hero 图建议 WebP 格式，1920×1080px
- 推荐使用 Squoosh (squoosh.app) 或 TinyPNG 压缩
